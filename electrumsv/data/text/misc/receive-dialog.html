The Expected payment dialog

Provisional in-wallet help

The Expected payment dialog is used to both view and edit existing incoming payments for the selected account. The Receive tab is used to declare to the wallet that the user expects an incoming payment to be received at some point. Unless the user has declared this to the wallet, the wallet will not look for the payment and will never know about it. For this reason the user must click the Create button before a payment destination is reserved and made available to them to give out.

Sharing payment details

Once an expected payment has been created, it is useful to share it with whoever is going to pay it. There are several ways this can be done.

Copy payment link

This button copies a bitcoin: style link. This link includes all the relevant details including the description, and the BSV amount. This is a BIP21 style link.

Copy the payment destination

This can either be done by selecting the text in the Payment destination field and copying it the same way you would with any other selected text, or you can click the copy button on the right hand side of the field. This copies the shown payment destination whether it is an address or a BIP276-encoded script in the case of accounts that cannot use addresses.

The QR code

The QR code encodes the payment link, and using a device that can scan the QR code, it is expected that the device can process that link.

Payment fields

Account

Because it is possible for an expected payment dialog to be open for a different account other than the currently selected one, we show the account that the expected payment belongs to.

Payment destination

This is the assigned key for the given payment. If the account is the type that uses addresses it will be shown as an address, otherwise it will be shown as a BIP276-encoded script.

Description (required)

The Description field describes the expected payment so that the user can distinguish it from any of their other payments.

Requested amount (optional)

The Requested amount field if empty, represents an incoming payment of unknown value or perhaps the user cannot be bothered to fill out a value. If a value is provided, then the expected payment will not be closed out until either sufficient payments are received to meet that value, or the expiry period has passed. Address reuse is considered to lower privacy, so it is recommended that anyone paying pay as one single transaction and not multiple, unless a separate expected payment is created for each transaction.

If fiat is enabled for the wallet two fields will be shown here, both BSV and the selected fiat currency. If fiat is not enabled, then only the BSV field will be shown here.

Request expires

This currently describes the expiry period of the payment. It cannot currently be edited.
